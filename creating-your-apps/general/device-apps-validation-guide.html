Aperçu

À un moment donné du développement de vos applications à partir de Brightcove Beacon Experience, vous recevrez des versions de test pour vous assurer que les applications se comportent comme vous le souhaitez. Ce document fournira des informations sur chacun des éléments suivants :

  • Catégories de test: Définir les grands domaines de test.
  • Des astuces: Suggestions et idées à suivre lors de la réalisation des tests.
  • Rapports: La procédure à suivre si vous rencontrez des problèmes lors de vos tests.

Avant de commencer

Assurez-vous que vous êtes configuré pour des tests appropriés. Assurez-vous des points suivants :

  • Les testeurs disposent de tous les appareils de test nécessaires.
  • Par souci de compatibilité, les appareils utilisés pour les tests ne doivent pas avoir plus de trois ans. Plus loin dans ce document, les versions minimales requises seront spécifiées pour chaque plate-forme.

Catégories de test

Brightcove recommande de tester les domaines suivants et de répondre aux questions :

  • Conception et fonctionnalités :
    • L'expérience correspond-elle aux maquettes convenues ?
    • Toutes les fonctionnalités demandées sont-elles disponibles ?
  • Mise en page et contenu :
    • Les pages sont-elles configurées comme prévu ?
    • Le contenu attendu apparaît-il dans chacune des pages ?
  • Configuration de la monétisation :
    • Les annonces s'affichent-elles comme prévu ?
    • Les abonnements s'affichent-ils pour les actifs correspondants ?
    • Les utilisateurs peuvent-ils acheter et avoir droit au contenu comme prévu ?

Des astuces

Veuillez tenir compte des conseils suivants lorsque vous effectuez votre validation :

  • Premier passage: Choisissez UNE des plates-formes et passez en revue les différents domaines de test sur cette plate-forme. Commencer avec Conception et fonctionnalités, puis passez à Mise en page et contenu et enfin à Configuration de la monétisation essai.
  • Des séries de tests supplémentaires: Lorsque vous avez terminé le premier passage pour chaque zone, passez aux autres plates-formes disponibles. Si un problème s'applique à plusieurs plates-formes, veuillez le noter dans tous les billets signalés précédemment.
  • Tester les correctifs: Les correctifs pour les problèmes signalés seront dans l'un de ces trois domaines :
    • L'expérience Brightcove Beacon (aucune mise à jour d'application nécessaire)
    • Dans la ou les applications
    • Le backend Brightcove Beacon

    Si le correctif est dans l'application, une nouvelle version sera fournie avec les détails de ce qui a été corrigé. Pour les autres zones, Brightcove fournirait des détails dans le ticket sur ce qui a été corrigé. Pour accélérer la correction et le nouveau test, il est très important que les tickets fournissent des détails sur le problème et les étapes pour aider au nouveau test.

Rapports

Brightcove vous recommande de signaler les problèmes au fur et à mesure que vous les rencontrez pour que les commentaires continuent d'être transmis à Brightcove et que les problèmes soient résolus et traités pour validation le plus rapidement possible. Veuillez signaler un problème/ticket par problème et fournir les informations suivantes dans chaque problème/ticket :

  • Plateforme testée
  • La version de l'application dans laquelle le problème a été détecté
  • Étapes à reproduire
  • Veuillez inclure des captures d'écran ou des films d'écran pour aider à accélérer l'identification du problème et fournir une résolution
