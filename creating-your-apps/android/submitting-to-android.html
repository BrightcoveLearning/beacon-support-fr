introduction

Ce document vous aidera à soumettre votre application Brightcove Beacon au Google Play Store. Les trois premières sections de ce document, à l'exception de celle-ci, répertorient les informations/actifs dont vous aurez besoin lorsque vous commencerez à terminer le processus de soumission.

À la suite de ces listes, vous trouverez des conseils étape par étape dans le processus de soumission. Après avoir créé votre compte, vous accéderez au tableau de bord où vous fournirez les informations nécessaires à la soumission. Le tableau de bord apparaît comme suit et les zones dans lesquelles vous devrez saisir des informations sont surlignées en jaune :

Android Mobile et Android TV

Les informations/actifs suivants seront requis si vous soumettez pour Android Mobile ou Android TV.

  • Aperçu du magasin : https://developer.android.com/distribute
  • Présentation du programme pour développeurs : https://developers.google.com/
  • Familiarisez-vous avec le vaste À faire liste: Liste de contrôle de lancement
  • Détails du magasin : https://play.google.com/apps/publish/signup/ (nécessite Google Play Developer Program - Adhésion, 25 $ une fois)
  • Délai suggéré pour l'approbation : Une semaine
  • Vous devez configurer les connexions de l'application de l'appareil pour le contrôle qualité en magasin
  • Assurez-vous que votre application TV est conforme à celle de Google Qualité de l'application TV normes.
  • Une clé de signature doit être créée qui sera utilisée pour signer l'application. Les informations suivantes sont requises pour créer la clé de signature :
    • Mot de passe
    • Prénom et nom
    • Nom de l'unité organisationnelle
    • Nom de l'organisation
    • Nom de la ville / localité
    • Nom de l'État / Province
    • Code de pays à deux lettres pour cet appareil
    REMARQUE: La clé de signature pour les applications mobiles et TV DOIT être la même si elles partagent une fiche Play Store à l'aide de plusieurs APK.

Informations requises pour Android Mobile

Il s'agit d'exigences spécifiques pour Android Mobile en plus des exigences communes énumérées ci-dessus.

  • Titre de l'application - 50 caractères
  • Description courte - 80 caractères
  • Description complète - 4000 caractères
  • Noter : Toutes ces métadonnées doivent être traduites dans toutes les langues si elles sont disponibles.
  • ID d'application - exemple - com.myco.tvcable
  • Keystore pour signer le paquet
  • Type d'application
  • Catégorie
  • Captures d'écran (voir les Éléments graphiques, captures d'écran et vidéo document pour plus d'informations)
    • Standard - Anglais (Royaume-Uni) - en-GB
    • JPEG ou PNG 24 bits (pas d'alpha). Longueur minimale de la page : 320px. Longueur maximale des pages : 3840px.
    • Vous devez télécharger au moins une capture d'écran de 7 pouces et une capture d'écran de 10 pouces pour permettre à votre application d'apparaître dans le Développé pour les tablettes liste dans le Play Store. Si vous avez déjà téléchargé des captures d'écran, déplacez-les dans la zone appropriée ci-dessous.
  • Icône haute résolution
    • Standard - Anglais (Royaume-Uni) - en-GB
    • 512x512
    • PNG 32 bits (avec alpha)
  • Graphique caractéristique
    • Standard - Anglais (Royaume-Uni) - en-GB
    • 1024b x 500h
    • JPG ou PNG 24 bits (sans alpha)
  • Icône du logo
    • Icône d'application 192 x 192 (un carré et un rond)
    • Client pour configurer les connexions d'application pour l'assurance qualité en magasin

Informations requises pour Android TV

Il s'agit d'exigences spécifiques pour Android TV en plus des exigences communes énumérées ci-dessus.

  • Package APK créé avec une signature générée
  • Titre
  • brève description
  • Description complète
  • Captures d'écran
    • JPEG ou PNG 24 bits (pas d'alpha). Jusqu'à 8 peuvent être téléchargés
    • 1280x720
  • Icône haute résolution
    • 512x512
    • PNG 32 bits (avec alpha)
  • Bannière
    • 320x180
    • PNG 24 bits
  • IC_lanceur
    • 256x256
    • PNG 24 bits
  • Graphique caractéristique
    • 1024 l x 500 h
    • JPG ou PNG 24 bits (sans alpha)
  • Client pour configurer les connexions d'application pour l'assurance qualité en magasin

Démarrer dans le tableau de bord

  1. Naviguez jusqu'au Page d'inscription à la console Google Play et créez votre compte. Cela comprendra le paiement de frais de 25 $. Après avoir terminé le processus d'inscription, vous verrez le Console de jeu Google:
  2. Clique le CRÉER UNE APPLICATION bouton.
  3. Fournir le Langage par défaut et application Titre.
  4. Clique le Créer bouton. Vous serez transféré dans le Liste de magasins section.

Compléter Liste de magasins informations

Les Liste de magasins section est assez longue. Les différentes sections seront affichées dans différentes étapes.

  1. Entrer le Brève description et Description complète.
  2. Fournissez les actifs graphiques comme demandé dans le formulaire. Vous n'avez PAS besoin d'entrer des images pour PORTER OS.
    Les Bannière TV et Image stéréoscopique à 360 degrés Daydream et pas important pour Brightcove Beacon. La vidéo promotionnelle YouTube n'est nécessaire que si vous utilisez YouTube pour promouvoir votre application.
  3. Complétez le Catégorisation section.
  4. Complétez le Détails du contact section.
  5. Clique le ENREGISTRER LE BROUILLON bouton.

Compléter Versions d'applications informations

Dans ce Versions d'applications section, vous allez créer un Piste fermée pour permettre le test de votre application avant sa mise en ligne. La piste fermée vous permet de choisir un certain nombre de personnes pour tester votre application.

  1. Clique le Versions d'applications lien dans la navigation de gauche.
  2. Clique le CRÉER UNE PISTE FERMÉE lien, surligné en jaune ci-dessous.
    Après avoir testé et vous êtes prêt à publier votre application, vous utilisez le Versions d'applications section à nouveau, mais sélectionnez Piste de fabrication pour télécharger l'application que vous allez diffuser en direct.
  3. Saisissez un nom pour votre circuit fermé.
  4. Clique le CRÉER UNE PISTE bouton.
  5. Pour créer la piste, vous devrez effectuer les tâches suivantes :
    1. Téléchargez l'APK (qui vous a été fourni par Brightcove)
    2. Fournissez un nom de version, par exemple 2.1.
    3. Fournissez des notes de version, dans autant de langues que vous prenez en charge.
    4. Enregistrez ou révisez les informations fournies.

Compléter Évaluation du contenu informations

Déplacez-vous dans le Évaluation du contenu section en utilisant la navigation de gauche. Dans cette section, vous remplissez un questionnaire et, en fonction de vos réponses, une note sera attribuée à votre application et à son contenu. Pour des informations complètes, consultez le Évaluations de contenu pour les applications et les jeux document.

Compléter Contenu de l'application informations

Déplacez-vous dans le Contenu de l'application section en utilisant la navigation de gauche. Dans cette section, vous fournissez des informations sur :

  • Votre politique de confidentialité souhaitée
  • Si des publicités seront utilisées dans votre contenu
  • S'il y a des parties restreintes de votre application
  • Le public cible de votre application

Compléter Prix et distribution informations

Dans cette section, vous fournissez des informations sur :

  • Si l'application est gratuite ou non
  • Géo-restrictions
  • Appareils Android sur lesquels l'application fonctionnera
  • Consentement sur la manière dont l'application sera gérée par Google

Les Prix et distribution section est assez longue. Les différentes sections seront affichées dans différentes étapes.

  1. Clique le Prix et distribution lien dans la navigation de gauche.
  2. Cliquez sur le bon bouton pour déterminer si votre application sera gratuite ou non. Notez que l'installation de l'application est gratuite ou non. Si vous allez gagner des revenus par des publicités ou des abonnements, et non par des frais d'installation, cliquez sur LIBRE .
  3. Configurer le Des pays section pour refléter toutes les restrictions géographiques que vous avez ou n'avez pas.
  4. Vous n'avez rien à faire dans le Catégories d'appareils section.
  5. Vous n'avez rien à faire dans le Programmes utilisateur section.
  6. Lisez les informations appropriées pour chaque question dans le Consentement section et cochez ou décochez les cases.

Ajout d'abonnements

Si vous ajoutez des abonnements à votre application Android, veuillez consulter le Ajouter des abonnements à Android document pour des instructions complètes.
