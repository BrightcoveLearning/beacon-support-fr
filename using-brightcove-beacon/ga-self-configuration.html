introduction

Le processus de configuration comprend les tâches suivantes :

  • Créez quatre propriétés Google Analytics.
  • Créez un ensemble de dimensions personnalisées et une statistique personnalisée pour chaque propriété.
  • Créez quatre conteneurs Google Tag Manager.
  • Configurez Google Tag Manager pour chaque conteneur.
  • Configurez Firebase pour faciliter la collecte d'analyses avec iOS et Android.
  • Installez des modèles dans Google Data Studio pour afficher vos données d'analyse.

Outils requis

Brightcove utilise divers outils pour mettre en œuvre la collecte, la manipulation et l'affichage des données analytiques à l'aide de Google Analytics. Ces outils sont :

  • Google Analytics - Une plate-forme d'analyse Web proposée par Google dans le cadre de sa marque Google Marketing Platform.
  • Gestionnaire de balises Google - Tout d'abord, comprenez que les balises sont des extraits de code qui sont ajoutés à un site pour collecter des informations et les envoyer à des tiers. Les balises sont utilisées pour de nombreuses raisons, notamment le suivi du défilement, le suivi des soumissions de formulaires, la réalisation d'enquêtes, etc. Avec Google Tag Manager, vous pouvez « gérer et mettre à jour les balises rapidement et prendre des décisions plus éclairées plus rapidement. Tag Manager s'intègre à toutes les balises Google et tierces, et vous donne un contrôle total sur le moment et la manière dont vos balises se déclenchent."
  • Base de feu - Firebase est une plateforme développée par Google pour la création d'applications mobiles et web. Firebase comprend le Base de données en temps réel Firebase , qui est une base de données hébergée dans le cloud.
  • Studio de données Google - Google Data Studio est un "outil gratuit qui transforme vos données en tableaux de bord et rapports informatifs, faciles à lire, faciles à partager et entièrement personnalisables".

Obtenir des modèles et des fichiers de configuration

Il existe un certain nombre de fichiers/URL dont vous avez besoin pour terminer avec succès votre configuration d'analyse pour Brightcove Beacon. Les fichiers/URL sont détaillés dans les puces suivantes. Vous devez contacter votre responsable d'intégration Brightcove pour obtenir ces fichiers/URL. Vous allez recevoir:

  • URL vers trois configurations de tableau de bord Google Analytics prédéfinies, une pour le mobile, le Web et la télévision intelligente.
  • Quatre fichiers JSON qui configurent chacun des quatre conteneurs créés dans Google Tag Manager.
  • URL vers trois modèles Google Data Studio prédéfinis, un pour mobile, web et smart tv.

Les instructions d'installation/d'utilisation de chacun des fichiers/URL apparaissent dans les sections appropriées plus loin dans ce document.

Vous devrez obtenir ces fichiers/URL AVANT de commencer le processus de configuration.

Configurer Google Analytics

Dans cette section du document, vous allez configurer Google Analytics, qui comprend les tâches suivantes :

  • Créez quatre propriétés Google Analytics.
  • Créer une vue mobile pour les propriétés iOS et Android
  • Créez un ensemble de dimensions personnalisées pour chaque propriété.
  • Créez une seule métrique personnalisée pour chaque propriété.
  • Importez trois configurations de tableau de bord créées par Brightcove, une pour le mobile, le Web et la smart tv.

Créer quatre propriétés

Vous devez créer ces quatre propriétés :

  • Balise Android
  • Balise iOS
  • Balise STV (Smart TV)
  • Balise Web

Les étapes suivantes vous guident dans la création du Balise Android biens. Vous devrez créer les autres en suivant le même processus.

  1. Connectez-vous à votre compte Google Analytics.
  2. Clique le Administrateur (équipement) icône en bas à gauche de l'écran.
  3. En haut de la colonne du milieu, cliquez sur Créer une propriété.
  4. Dans le Configuration de la propriété , fournir le Nom de la propriété , pour la première propriété, utilisez Balise Android.
  5. Fournir le Fuseau horaire des rapports.
  6. Fournir le Monnaie.
  7. Clique le Montrer les options avancées relier.
  8. Allumer Créez une propriété Universal Analytics.
  9. Entrez l'URL de votre site Web.
  10. Clique le Créer une propriété Universal Analytics uniquement bouton radio.
  11. Assurez-vous que les informations de votre propriété sont remplies de la manière suivante :
  12. Cliquez sur Prochain.
  13. Complétez le Informations d'affaires formulaire avec vos données.
  14. Cliquez sur Créer.
  15. Lisez tous les détails que vous souhaitez sur votre nouvelle propriété.
  16. Cliquez sur la flèche de retour, mise en évidence dans la capture d'écran suivante :
  17. Juste en dessous du Créer une propriété vous avez cliqué plus tôt, cliquez sur la flèche vers le bas pour vous assurer que votre propriété a été correctement créée.
  18. Effectuez les étapes suivantes pour créer les trois autres propriétés requises :
    1. Balise iOS
    2. Balise STV
    3. Balise WEB

Créer une vue mobile pour iOS et Android

Pour les plates-formes mobiles, iOS et Android, vous devez créer une vue mobile dans chacune de ces propriétés. Vous n'avez pas besoin de suivre les étapes de cette section si vous n'avez pas d'applications mobiles.

  1. Cliquez sur la liste déroulante (surlignée en jaune dans la capture d'écran) pour voir tous vos comptes, propriétés et vues Google Analytics.
  2. Dans la liste déroulante à trois volets, sélectionnez :
    • Votre compte Brightcove Beacon
    • Les Balise Android biens
    • Les Toutes les données du site Web vue
  3. Clique le Administrateur (équipement) icône en bas à gauche de l'écran.
  4. En haut de la troisième colonne, cliquez sur Créer une vue.
  5. Remplissez le à partir de ces étapes :
    1. Clique le Application mobile bouton pour le type de données à suivre.
    2. Donnez un nom à votre vue. Un nom logique serait Toutes les données mobiles.
    3. Cliquez sur Créer une vue.
  6. Remplacez la vue par la vue mobile nouvellement créée.
  7. Suivez les mêmes étapes dans cette section du document pour créer une vue mobile pour la propriété iOS.

Lorsque vous travaillez avec vos comptes, assurez-vous que lorsque vous sélectionnez un compte/une propriété/affichez vos propriétés mobiles, utilisez le Toutes les données mobiles vue et les comptes Web et STV utilisent la valeur par défaut Toutes les données du site Web , comme indiqué ici :

Créer des statistiques et des dimensions personnalisées

Maintenant, pour chaque propriété, vous devez créer un ensemble de dimensions personnalisées et une métrique personnalisée.

Cette section sera organisée de la manière suivante :

  • Les étapes seront montrées comment créer une seule dimension personnalisée.
  • Les étapes seront montrées comment créer une seule métrique personnalisée.
  • Un tableau indiquera les dimensions requises pour les propriétés Balise Android et Balise iOS.
  • Un tableau indiquera les dimensions requises pour les propriétés Balise STV et Balise WEB.
  • Un tableau montrera la seule métrique qui tout les quatres propriétés doivent avoir.

Vous devrez ensuite créer les dimensions et la métrique pour chacune des quatre propriétés.

Création d'une dimension personnalisée

  1. Sélectionnez l'une de vos propriétés dans le sélecteur de compte/propriété/vue pour laquelle vous souhaitez créer les dimensions personnalisées.
  2. Clique le Administrateur (équipement) icône en bas à gauche de l'écran.
  3. Près du bas de la colonne du milieu, sous Définitions personnalisées , Cliquez sur Dimensions personnalisées
  4. Cliquez sur NOUVELLE DIMENSION PERSONNALISÉE.
  5. Pour le Nom Entrer Numéro d'épisode.
  6. Du Portée liste déroulante, sélectionnez Frappé.
  7. Quitter la cote actif.
  8. Cliquez sur Créer.
  9. Dans l'écran d'exemple de code, cliquez sur Terminé.
  10. Vous verrez votre nouvelle dimension dans une liste, comme illustré ici :
    Il est crucial que le Indice correspond au nom de la dimension comme indiqué dans les tableaux ci-dessous. Pour cela, il suffit de créer les dimensions dans l'ordre indiqué dans les tableaux.
  11. Pour créer une autre dimension, cliquez sur NOUVELLE DIMENSION PERSONNALISÉE.

Créer une métrique personnalisée

  1. Sélectionnez l'une de vos propriétés dans le sélecteur de compte/propriété/vue pour laquelle vous souhaitez créer la métrique personnalisée.
  2. Clique le Administrateur (équipement) icône en bas à gauche de l'écran.
  3. Près du bas de la colonne du milieu, sous Définitions personnalisées , Cliquez sur Métriques personnalisées
  4. Cliquez sur NOUVELLE MÉTRIQUE PERSONNALISÉE.
  5. Pour le Nom Entrer Temps de visionnage.
  6. Du Portée liste déroulante, sélectionnez Frappé.
  7. Pour le Type de formatage , sélectionnez Temps.
  8. Quitter la cote actif.
  9. Cliquez sur Créer.
  10. Dans l'écran d'exemple de code, cliquez sur Terminé.
  11. Vous verrez votre métrique nouvellement créée dans une liste.
Il s'agit de la seule métrique personnalisée que vous devrez créer pour chacune des quatre propriétés.

Configuration de Google Gestionnaire de balises

Dans cette section du document, vous effectuerez les tâches suivantes :

  • Créez quatre conteneurs, un pour chaque propriété créée précédemment.
  • Importez les fichiers JSON de configuration qui vous ont été envoyés par e-mail font partie du Obtenir les modèles et les fichiers de configuration rubrique ci-dessus.
  • Saisissez votre identifiant de suivi, copié depuis votre compte Google Analytics, dans chacun des conteneurs' Code de suivi variable.
  • Publiez vos conteneurs.

Créer des conteneurs

  1. Consultez la documentation de Google pour Configurer et installer Tag Manager.

  2. Dans Gestionnaire de balises , créez un compte ou utilisez un compte existant. Pour créer un nouveau compte, sélectionnez le Créer un compte bouton.

    Par défaut, un nouveau conteneur est créé pour un nouveau compte. Vous créerez des conteneurs supplémentaires dans chaque compte.

  3. Saisissez des valeurs pour votre nouvelle page de compte :

    • Nom du compte: Le nom de votre entreprise, ou peut-être balise.
    • Pays: Pays d'origine de votre entreprise.
    • Nom du conteneur: Balise WEB Vous allez créer un conteneur pour chaque propriété.
    • Plateforme cible: la toile
  4. Assurez-vous que les informations de votre conteneur sont remplies comme suit :
  5. Cliquez sur Créer.
  6. Cliquez sur d'accord pour fermer la boîte de dialogue qui s'ouvre.
  7. Du Espace de travail , Cliquez sur Administrateur.
  8. Cliquez sur le signe plus pour ajouter un autre conteneur.
  9. Remplissez le formulaire et cliquez Créer pour créer le iOS récipient.
  10. Assurez-vous que les quatre conteneurs suivants sont créés avec le bon Plateforme cible.
    Nom du conteneur Plateforme cible
    Balise Android Android
    Balise iOS iOS
    Balise STV la toile
    Balise WEB la toile
  11. Du Espace de travail , utilisez la liste déroulante du compte en haut de la page pour vous assurer que tous les conteneurs ont été créés.
  12. À ce stade, il est essentiel que vous preniez une capture d'écran de cette liste déroulante. Les ingénieurs de Brightcove Beacon ont besoin de ces données, à savoir les noms des conteneurs et les GTM IDENTIFIANT. Vous enverrez ce fichier à votre responsable d'intégration avec les autres fichiers téléchargés, comme expliqué dans le Publier chaque conteneur section qui vient plus loin dans ce document. Importer des fichiers de configuration

Effectuez les étapes suivantes pour importer les fichiers de configuration de vos conteneurs :

  1. Du Espace de travail , Cliquez sur Administrateur.
  2. Vérifiez quel conteneur vous êtes en train de configurer, puis cliquez sur Conteneur d'importation.
  3. Cliquez sur Choisir le fichier conteneur , et sélectionnez le fichier JSON approprié pour ce conteneur.
  4. Pour Choisir l'espace de travail , Cliquez sur Existant , puis dans la boîte de dialogue qui s'ouvre, sélectionnez Espace de travail par défaut.
  5. Pour Choisissez une option d'importation , clique le Écraser bouton radio.
  6. Assurez-vous de voir les informations sur les balises, les déclencheurs, les variables et les modèles.
  7. Cliquez sur Confirmer , et vous serez ramené au Espace de travail..
  8. Dans le Espace de travail , cliquer sur Mots clés , Déclencheurs et Variables pour confirmer que l'importation a réussi.
  9. Vous devez maintenant importer le bon fichier JSON pour chaque conteneur.

Saisir l'identifiant de suivi

Vous allez maintenant copier depuis Google Analytics l'ID de suivi de chaque propriété, puis le coller dans le conteneur correspondant.

  1. Connectez-vous à Google Analytics.
  2. Utilisez le sélecteur de compte/propriété en haut de la page pour choisir le Balise Android biens.
  3. Cliquez sur Administrateur.
  4. Du Biens colonne premier clic Informations de suivi alors Code de suivi.
  5. Copiez le Identifiant de suivi.
  6. Revenez à Google Tag Manager.
  7. Assurez-vous que vous modifiez le Balise Android récipient.
  8. Cliquez sur Variables de la navigation de gauche.
  9. Clique le Code de suivi variable.
  10. Collez l'ID de suivi copié de Google Analytics dans le Valeur champ de formulaire.
  11. Vous devez maintenant coller l'ID de suivi correspondant dans le Code de suivi variable dans chaque conteneur.

Publier chaque conteneur

Plus tard dans ces étapes, vous publierez vos conteneurs. Pour iOS et Android, vous devrez télécharger des fichiers de configuration et demander à votre équipe de développement d'applications d'ajouter ces fichiers à l'application respective.

La dernière étape que vous devez faire dans Google Tag Manager est de soumettre chaque conteneur pour publication. Les étapes suivantes vous guident tout au long de ce processus :

  1. Retournez dans votre espace de travail.
  2. Notez le conteneur dans lequel vous vous trouvez.
  3. En haut à droite de l'interface utilisateur, cliquez sur Soumettre.
  4. Assurez-vous que le Publier et créer une version le bouton est sélectionné.
  5. Si vous le souhaitez, ajoutez un nom de version et une description de version.
  6. Cliquez sur Publier.
  7. Si vous n'avez pas fourni de nom de version et de description, on vous le demandera à nouveau. Sautez si vous êtes sûr de ne pas en vouloir.
  8. Assurez-vous que les informations renvoyées sur votre conteneur sont correctes.
  9. Pour les conteneurs iOS et Android: Cliquez sur Télécharger en haut à droite de l'UI pour télécharger le fichier associé. Ce fichier devra être envoyé à votre responsable d'intégration.
  10. Pour les conteneurs iOS et Android: Contactez votre responsable d'intégration et envoyez-lui ces fichiers téléchargés, accompagnés de la capture d'écran de vos conteneurs et des identifiants correspondants. Les fichiers seront ajoutés à vos applications correspondantes.
  11. Assurez-vous que tous les conteneurs sont publiés.
Pour réitérer, à ce stade, vous devez envoyer les fichiers téléchargés, ainsi que la capture d'écran montrant chaque nom de conteneur et associé GTM ID, à votre responsable d'intégration. Ces fichiers seront ensuite acheminés vers le personnel d'ingénierie de Brightcove Beacon et pour configurer vos applications.

  6. Choisissez ou fournissez un compte Google Analytics.
  7. Cliquez sur Créer un projet.
  8. Assurez-vous d'être renvoyé au Aperçu du projet page. Notez également les boutons de création d'applications iOS et Android (les flèches rouges indiquent leurs positions).
  9. Si vous utilisez une application iOS, cliquez sur le bouton de l'application iOS et suivez les cinq étapes pour ajouter Firebase à votre application iOS. Pour terminer les étapes, vous aurez besoin d'un développeur iOS pour vous aider. Le document Google suivant fournit des détails sur les étapes :
    • Ajouter Firebase à votre projet iOS.
  10. Si vous utilisez une application Android, cliquez sur le bouton de l'application Android et suivez les quatre étapes pour ajouter Firebase à votre application Android. Pour terminer les étapes, vous aurez besoin d'un développeur Android pour vous aider. Le document Google suivant fournit des détails sur les étapes :
    • Ajouter Firebase à votre projet Android.

Après avoir ajouté les fichiers Goggle Tag Manager et Firebase, comme détaillé dans les deux dernières sections de ce document, vous devriez pouvoir vérifier les événements dans Firebase lui-même. Notez que Google Tag Manager agit ici comme un intermédiaire, en envoyant ces informations à Google Analytics en fonction des données transmises à Firebase.
