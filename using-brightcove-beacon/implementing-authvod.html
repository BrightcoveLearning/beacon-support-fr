    Mise en œuvre d'AuthVod

    Dans cette rubrique, vous apprendrez comment créer un package de monétisation AuthVod.

    Comprendre AuthVod

    AuthVod (AuthVod (Authenticated Video On Demand) est un package de monétisation dans lequel les spectateurs doivent s'authentifier, c'est-à-dire entrer un nom d'utilisateur et un mot de passe, à l'application Brightcove Beacon pour regarder les vidéos incluses dans le package. Les vidéos peuvent afficher ou non des publicités, selon la façon dont vous configurez ce package.

    Consultez le document Présentation des options de monétisation pour plus d'informations sur AuthVod, ainsi que sur tous les packages de monétisation que vous pouvez utiliser avec Brightcove Beacon.

    Présentation de la configuration d'un package AuthVod

    Voici les étapes de haut niveau de la configuration d'un package AuthVod :

    1. Dans Brightcove Beacon, créez un package non publié et gratuit.
    2. Si des annonces sont souhaitées, fournissez un fournisseur d'annonces lors de la création du package.
    3. Dans Video Cloud Studio, ajoutez des données aux champs personnalisés des vidéos pour les affecter au package AuthVod.
    4. Dans Brightcove Beacon, ingérer les vidéos et vider le cache.

    Création d'un package dans Brightcove Beacon

    Voici les étapes détaillées à suivre dans Brightcove Beacon pour créer un package AuthVod :

    1. Cliquez sur l'onglet Commerce situé en haut de Brightcove Beacon.
    2. Sélectionnez Packages dans la navigation de gauche.
    3. Cliquez sur le bouton Ajouter un nouveau package .
    4. Sous l'onglet Package , indiquez le nom interne et assurez-vous de laisser le statut comme non publié.
    5. Si vous souhaitez utiliser de la publicité, vous pouvez cliquer sur le bouton radio Oui pour Publicité , puis choisir un fournisseur de publicités dans la liste déroulante.
    6. Dans l'onglet Données textuelles , entrez le nom visible pour les visionneuses et un titre sur le package.
    7. Si vous le souhaitez, fournissez une image qui s'affichera avec les informations du paquet.
    8. Sous l'onglet Streams , entrez des valeurs pour le nombre maximal de périphériques pouvant être diffusés en continu en même temps, ainsi que le nombre maximal de périphériques pouvant disposer d'une connexion active.
    9. Si vous souhaitez un contrôle plus fin de la disponibilité du package, cliquez sur le bouton radio Oui pour Advanced Streams ?, puis remplissez le formulaire avec des options de disponibilité plus fines.

    Ajouter des vidéos au package dans Studio

    Pour ajouter du contenu au package, vous devez utiliser Video Cloud Studio pour attribuer des valeurs à des champs personnalisés. Les deux champs personnalisés auxquels vous devez affecter des valeurs pour un package AuthVod sont les suivants :

    • beacon.rights..NomPackageName
    • beacon.rights..type

    Les étapes ci-dessous détaillent le processus d'attribution de valeurs à ces champs personnalisés.

    Pour plus d'informations sur l'utilisation des champs personnalisés de Video Cloud avec Brightcove Beacon, consultez le document Creating Video Cloud Custom Fields .

    Pour chaque vidéo que vous souhaitez être dans le package, procédez comme suit :

    1. Connectez-vous à Video Cloud Studio.
    2. Dans le module Media, cliquez sur le nom de la vidéo pour afficher les propriétés de cette vidéo.
    3. Cliquez sur le bouton CHAMPS PERSONNALISÉS' Modifier.
    4. Pour la beacon.rights.<compteur > .PackageName, entrez le nom de votre package AuthVod créé précédemment.
    5. Pour la beacon.rights.<compteur > .type, entrez SVOD.

    La capture d'écran suivante montre un exemple de valeurs réelles utilisées dans les champs personnalisés. Dans ce cas, le compteur est zéro (surligné en rouge) et le nom du paquet est AuthVod Test (surligné en jaune). Rappelez-vous lors de l'implémentation AuthVod, beacon.rights.<counter > .type est toujours défini sur SVOD.

    Incorporer des vidéos dans Brightcove Beacon

    Les modifications apportées aux vidéos dans Video Cloud Studio doivent être ingérées dans Brightcove Beacon. Cela se fait automatiquement toutes les heures, mais vous pouvez également ingérer manuellement à la demande. Les grandes étapes seront détaillées ci-dessous.

    Pour plus de détails sur l'ingestion de vidéos, consultez le document Forcer une synchronisation .
    1. Dans Brightcove Beacon, dans le menu supérieur, cliquez sur l'onglet Outils (icône clé).
    2. Dans la navigation de gauche, assurez-vous que l'option Ingestion est sélectionnée (c'est l'option par défaut).
    3. Cliquez sur le bouton Mettre à jour les vidéos Brightcove .
    4. Une fois l'ingestion terminée, vous pouvez vérifier que vos vidéos ont été attribuées à votre package AuthVod en cliquant d'abord dans le menu supérieur sur l'onglet Commerce .
    5. Dans la navigation de gauche, assurez-vous que l'option Packages est sélectionnée (c'est l'option par défaut).
    6. Cliquez sur le nom de votre package AuthVod.
    7. Cliquez sur l'onglet Contenu pour voir les ressources affectées à ce package AuthVod. Un exemple de cet onglet est illustré ici :
    8. Pour commencer à effacer le cache, dans le menu supérieur, cliquez sur l'onglet Outils (icône clé).
    9. Dans la navigation de gauche, cliquez sur l'option Cache .
    10. Cliquez sur le bouton Purge du cache .

    Désormais, tous les spectateurs authentifiés auprès de l'application Brightcove Beacon auront accès au contenu AuthVod. Si un spectateur tente de lire une vidéo alors qu'il n'est PAS authentifié, un message apparaît en bas à gauche de la vidéo pour l'inviter à se connecter, comme indiqué ici :
