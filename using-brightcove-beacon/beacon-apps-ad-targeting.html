introduction

Les informations côté client sont transmises des applications à SSAI, puis SSAI inclut ces données dans la balise VAST du client, ce qui permet ensuite aux clients de cibler les annonces en fonction de ces données. L'objectif est d'augmenter la valeur de l'inventaire publicitaire des clients, augmentant ainsi les revenus publicitaires de leur entreprise.

Vous pouvez configurer la manière dont ces données sont transmises à leur serveur publicitaire ou à leur fournisseur de publicité.

Avantages

Ciblage publicitaire amélioré

  • Identifiant de session utilisateur : nous générons un identifiant de session sur chaque appareil lorsqu'un utilisateur ouvre l'application. Cela vous permet de limiter le nombre d'annonces à cette session, d'exclure les annonces concurrentes d'être diffusées ensemble et d'effectuer une rotation séquentielle des annonces dans cette session. Ces capacités de ciblage sont à la fois nécessaires pour les annonces de vente directe et programmatiques
  • Plus de données pour les annonces programmatiques : Les applications mobiles et les applications CTV ne prennent pas en charge les cookies, qui constituent la méthode dominante de ciblage publicitaire basé sur l'utilisateur/l'appareil sur le Web. En incluant des paramètres tels que Device ID, vous pouvez transmettre ces informations précieuses aux acheteurs d'annonces programmatiques, qui amélioreront ensuite leurs capacités de ciblage publicitaire, conduisant à de meilleurs CPM.

Prise en charge de la répartition de l'inventaire publicitaire

Pour les applications qui s'exécutent sur Amazon, Roku et Samsung (aux États-Unis), vous devez effectuer une répartition 70/30 de l'inventaire publicitaire/des revenus avec la plate-forme. Les clients Beacon monétisent 70 % de l'inventaire publicitaire et la plate-forme obtient les 30 % restants de l'inventaire publicitaire. Avec ce ciblage publicitaire amélioré, vous pouvez utiliser un seul tag publicitaire dans toutes leurs applications et transmettre les paramètres de l'appareil à leur serveur publicitaire. À l'intérieur de leur serveur publicitaire, ils implémenteront la logique métier qui divise l'inventaire publicitaire par appareil. Les plates-formes nécessitent également de passer LAT - Limited Ad Targeting - pour Ad Inventory Split, qui est un paramètre que nous envoyons maintenant à votre serveur publicitaire.

Améliorations de la confidentialité des utilisateurs

  • LAT "Ciblage publicitaire limité" - permet aux utilisateurs d'applications Beacon de désactiver le ciblage publicitaire en fonction du comportement de l'utilisateur.
  • Les utilisateurs ont la possibilité de réinitialiser l'ID d'appareil unique que nous attribuons à l'appareil à tout moment.

Remarques

  • Cette fonctionnalité fonctionne sur toutes les plateformes/appareils iOS, Apple TV, Android, Roku, Fire TV, Web et Smart TV.
  • Le ciblage publicitaire Beacon Apps peut être utilisé avec VOD SSAI et Live SSAI.

Étapes de la VOD

Partie Video Cloud - VOD

  1. Créez une configuration d'annonce à partir du Page des paramètres d'annonce côté serveur dans Studio:
    1. Cliquez sur Créer une configuration d'annonce et sélectionnez VOD
    2. Ajoutez un nom pour la configuration de l'annonce<./li>
    3. Sélectionnez votre réponse publicitaire - VMAP ou Vast est recommandé. Si vous utilisez Vast, vous devrez ajouter des points de repère aux vidéos pour spécifier où les coupures publicitaires doivent apparaître.
    4. Collez votre tag d'emplacement publicitaire (de votre fournisseur d'annonces).
    5. Copiez votre identifiant de configuration publicitaire, car vous en aurez besoin pour la configuration de la publicité Beacon.

Partie balise - VOD

  1. Ouvrez l'interface utilisateur de Beacon Classic et accédez à l'onglet Publicité :
    1. Cliquez sur Ajouter une nouvelle configuration
    2. Nommez la configuration de l'annonce
    3. Sélectionner Brightcove SSAI
    4. Coller dans le ID de configuration d'annonce SSAI dans ID de configuration d'annonce VOD
  2. Une fois que vous avez configuré votre configuration publicitaire dans Studio et Beacon Classic, vous devez implémenter les paramètres côté client qui seront transmis au serveur publicitaire du client. Les paramètres côté client peuvent être transmis dans les API de lecture (fournies dans le point d'entrée VMAP/Manifest) - ceux-ci seront disponibles pour l'infrastructure SSAI sous la forme de {{url.xxx}} macros où le xxx correspond à la clé de paramètre qui est soumise dans l'URL de l'API de lecture.
    1. Saisissez les paramètres de ciblage des annonces dans le Cible publicitaire champ à Beacon. L'entrée sera une chaîne sous forme de requête. Exemple: d={PLATFORM}&os={OS}&homme={MFGR}&mo={MODEL}&appid={APP_ID}&lan={LOCALE}&a fait={RDID}&dt={IDTYPE}&lat={IS_LAT}&sid={SID}
    2. Vous pouvez configurer les clés avant le signe égal. Ils DOIT correspondent aux clés ajoutées au tag d'emplacement publicitaire SSAI.
    3. Ensuite, mettez à jour votre tag d'emplacement publicitaire dans le Paramètres SSAI Studio page. Voici un exemple de paramètres qui ont été saisis dans le ciblage publicitaire Beacon Classic et ajoutés à la balise publicitaire SSAI : d={PLATFORM}&os={OS}&appid={APP_ID}&a fait={RDID}&dt={IDTYPE}&lat={IS_LAT}&sid={SID}
    4. Ces paramètres sont traduits en ceci dans ce tag d'emplacement publicitaire : https://ads.brightcove.com/ads?tech=dfpadrules&dur=15&pre=3&d={{url.d}}&os={{url.os}}&appid={{url.appid}}&a fait={{url.did}}&dt={{url.dt}}&lat={{url.latl}}&sid={{url.sid}}

Étapes en direct

  1. Tout d'abord, construisez votre chaîne de paramètres de ciblage publicitaire, car vous en aurez besoin à la fois dans les configurations SSAI et Beacon. Vous pouvez réutiliser les paramètres pour la VOD ou sélectionner un ensemble différent.

    Exemple:

    ip={{client.ipaddress}}&num={{random.int32}}&d={{PLATFORM}}&os={{OS}}&a fait={{RDID}}&dt={{IDTYPE}}&lat={{IS_LAT}}
  2. Créer une configuration d'annonce à partir de la page des paramètres d'annonce côté serveur dans Studio:
    1. Cliquez sur Créer une configuration d'annonce et sélectionnez Habitent
    2. Ajouter un nom pour la configuration de l'annonce
    3. Sélectionner Vaste votre réponse à l'annonce
    4. Collez votre tag d'emplacement publicitaire, en ajoutant votre chaîne de requête des paramètres de ciblage des annonces.

      Exemple:

      https://ad-provider-host.com/path/to/ad-handler?ip={{client.ipaddress}}&num={{random.int32}}&d={{PLATFORM}}&os={{OS}}&a fait={{RDID}}&dt={{IDTYPE}}&lat={{IS_LAT}}
    5. Copiez votre ID de configuration publicitaire pour les configurations Beacon Advertising
  3. Ouvrez l'interface utilisateur Beacon Classic et accédez à l'onglet Publicité
    1. Cliquez sur Ajouter une nouvelle configuration
    2. Nommez la configuration de l'annonce
    3. Sélectionner Brightcove SSAI
    4. Collez l'ID de configuration d'annonce SSAI dans ID de configuration d'annonce en direct
  4. Collez votre chaîne de paramètres de ciblage publicitaire dans le Cible publicitaire champ.

    Exemple:

    ip={{client.ipaddress}}&num={{random.int32}}&d={{PLATFORM}}&os={{OS}}&a fait={{RDID}}&dt={{IDTYPE}}&lat={{IS_LAT}}
