introduction

La configuration de Beacon et Evergent pour qu'ils fonctionnent ensemble nécessite une certaine configuration dans les deux systèmes. Dans plusieurs cas, vous devez copier des informations d'un système à l'autre ou répliquer exactement des informations d'un système à l'autre. Si vous pouvez ouvrir les deux systèmes côte à côte dans des fenêtres de navigateur distinctes, cela facilitera l'exécution des étapes ci-dessous avec succès.

balise partie 1

Pour cette partie, vous aurez besoin d'une instance Beacon CMS (système de gestion des clients) configurée.

  1. Dans le Beacon CMS, configurez Evergent en tant que Boutique.
  2. Sur le Paramètres > Emplacement page de la boutique, définissez les pays dans lesquels le service sera disponible.
  3. Sur le Paramètres > Devises page de la boutique, définissez les devises pour chaque pays.

Evergent partie 1

Pour cette partie, vous devez configurer des BU (business units) dans le portail Evergent Go. Chaque BU Evergent sera mappée sur un pays ou une région spécifique du monde, et ceux-ci, ainsi que les devises qui leur sont associées, doivent correspondre à ce que vous avez entré dans le CMS Beacon.

  1. Configurez la devise de la BU.
  2. Configurer le pays où se situe la BU
  3. Configurez les passerelles de paiement et les moyens de paiement pour la BU :
    • Carte de crédit carte de débit
    • Portefeuilles électroniques
    • Achats intégrés (iOS, Google Play, Amazon Billing, Roku Pay)

balise partie 2

Les identifiants des BU Evergent et la cartographie des pays doivent être configurés par client Beacon. Cela se fait dans le service de paiement de monétisation, qui est un service principal appartenant à l'équipe Playback. (Le plan est que cette configuration passe à Jackie Pages d'ici la fin du T1 21). Ces configurations doivent être liées à un compte client Video Cloud spécifique utilisé pour Beacon.

balise partie 4

Un éternel sel pour ce client Beacon spécifique doit être configuré dans le service de paiement de monétisation.

balise partie 5

Le service de paiement de monétisation génère un point de terminaison "Beacon_Monetization_Api" (beacon.monetization.api.brightcove.com) qui doit être saisi dans l'instance Beacon CMS du client. Cela doit être entré dans la page Paramètres sous « Configuration de la passerelle de paiement » pour Evergent.

balise partie 6

  1. Créez des plans SVOD dans l'onglet Commerce de Beacon CMS. Définissez les règles de monétisation :
    1. Fréquence de facturation
    2. Période d'essai
    3. Politique de renouvellement
    4. Sélectionnez les magasins dans lesquels le plan s'affichera :
      • Evergent
      • Magasins d'applications (iOS, Google Play, Amazon, Roku)
    5. Définissez les appareils et les emplacements.
    6. Définissez les dates de disponibilité.
    7. Configurez les magasins d'applications :
      1. Configurez les dates de disponibilité.
      2. Configurez les appareils.
      3. Saisissez le prix et les devises.
      4. Créez les identifiants de magasin. Cet identifiant doit être copié et collé dans le sku du produit dans les magasins d'applications. En outre, il doit être saisi dans Evergent Go pour le plan SVOD correspondant.

Evergent partie 2

Dans Evergent Go, créez des forfaits SVOD correspondants. Il est important que les politiques de monétisation correspondent de manière identique à ce qui a été saisi dans le CMS de Beacon. Notez également que si vous souhaitez vendre des plans SVOD dans différentes devises, vous devez utiliser plusieurs BU Evergent. Les plans de chaque BU sont mappés à Beacon à l'aide de SKU qui sont saisis dans le « Evergent Store » dans le Beacon CMS.

balise partie 7

Configurez Beacon Evergent Store pour vos forfaits SVOD :

  1. Configurez la disponibilité.
  2. Configurez les appareils (pour le moment uniquement sur le Web).
  3. Configurez le prix et les devises. Les devises doivent correspondre aux plans des BU spécifiques qui utilisent cette devise. Par exemple, si vous avez un plan que vous vendez dans l'UE en utilisant des euros, l'identifiant Evergent doit provenir de la BU Evergent UE. Le champ ID Evergent est l'endroit où vous entrez le SKU Evergent.
