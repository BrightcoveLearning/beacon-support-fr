introduction

Les informations côté client sont transmises des applications à SSAI, puis SSAI inclut ces données dans la balise VAST du client, ce qui permet ensuite aux clients de cibler les annonces en fonction de ces données. L'objectif est d'augmenter la valeur de l'inventaire publicitaire des clients, augmentant ainsi les revenus publicitaires de leur entreprise.

Dans Studio et Beacon CMS, vous pouvez configurer la manière dont ces données sont transmises à leur serveur publicitaire ou fournisseur de publicité.

Avantages

Ciblage publicitaire amélioré

  • Identifiant de session utilisateur : Brightcove génère un identifiant de session sur chaque appareil lorsqu'un utilisateur ouvre l'application. Cela vous permet de limiter le nombre d'annonces à cette session, d'exclure les annonces concurrentes d'être diffusées ensemble et d'effectuer une rotation séquentielle des annonces dans cette session. Ces capacités de ciblage sont à la fois nécessaires pour les annonces de vente directe et programmatiques.
  • Plus de données pour les annonces programmatiques : Les applications mobiles et les applications CTV ne prennent pas en charge les cookies, qui constituent la méthode dominante de ciblage publicitaire basé sur l'utilisateur/l'appareil sur le Web. En incluant des paramètres tels que Device ID, vous pouvez transmettre ces informations précieuses aux acheteurs d'annonces programmatiques, qui amélioreront ensuite leurs capacités de ciblage publicitaire, conduisant à de meilleurs CPM.

Prise en charge de la répartition de l'inventaire publicitaire

Pour les applications qui s'exécutent sur Amazon, Roku et Samsung (aux États-Unis), vous devez effectuer une répartition 70/30 de l'inventaire publicitaire/des revenus avec la plate-forme. Cela signifie que les clients de Brightcove Beacon monétisent 70 % de l'inventaire publicitaire et que la plate-forme obtient les 30 % restants de l'inventaire publicitaire. Grâce à ce ciblage publicitaire amélioré, vous pouvez utiliser un seul tag publicitaire dans toutes leurs applications et transmettre les paramètres de l'appareil à votre serveur publicitaire. À l'intérieur de votre serveur publicitaire, vous implémenterez la logique métier qui divise l'inventaire publicitaire par appareil. Les plates-formes nécessitent également de passer LAT - Limited Ad Targeting - pour Ad Inventory Split, qui est un paramètre que nous envoyons maintenant à votre serveur publicitaire.

Améliorations de la confidentialité des utilisateurs

  • LAT "Ciblage publicitaire limité" - permet aux utilisateurs d'applications Beacon de désactiver le ciblage publicitaire en fonction du comportement de l'utilisateur.
  • Les utilisateurs ont la possibilité de réinitialiser l'ID d'appareil unique que nous attribuons à l'appareil à tout moment.

  • Cette fonctionnalité fonctionne sur toutes les plateformes/appareils iOS, Apple TV, Android, Roku, Fire TV, Web et Smart TV.

Implémenter les paramètres côté client

Que vous utilisiez le ciblage publicitaire avec VOD ou Live, vous devez créer la chaîne de requête qui sera utilisée avec les deux. Utilisez les instructions suivantes lors de la création de votre chaîne de requête :

  • Les paramètres côté client seront transmis au serveur publicitaire lors de la demande d'annonces.
  • Les paramètres côté client peuvent être transmis dans les API de lecture (fournies dans le point d'entrée VMAP/Manifest). Ceux-ci seront disponibles pour l'infrastructure SSAI sous la forme de {{url.xxx}} macros où le xxx correspond à la clé de paramètre.
  • Collez votre tag d'emplacement publicitaire, en ajoutant votre chaîne de requête des paramètres de ciblage des annonces: 
    variablename1={VALUEMACRO1}&variablename2={VALUEMACRO2}& ...
  • Exemple: 
    plarform={PLATFORM}&os={OS}&appid={APP_ID}&did={RDID}&dt={IDTYPE}&lat={IS_LAT}&sid={SID}

Assurez-vous d'avoir créé la chaîne de requête de ciblage publicitaire de votre choix, car vous en aurez besoin dans les étapes de mise en œuvre ci-dessous.

Étapes de la VOD

Étapes de Video Cloud - VOD

  1. Dans Video Cloud Studio, accédez à Admin → Paramètres d'annonce côté serveur. (Pour plus de détails, consultez le Mise en œuvre de la VOD SSAI document.)
  2. Du Créer une configuration d'annonce liste déroulante, sélectionnez VOD
  3. Ajoutez un nom pour la configuration de l'annonce.
  4. Sélectionnez votre réponse publicitaire - VMAP ou VAST est recommandé. Si vous utilisez VAST, vous devrez ajouter des points de repère aux vidéos pour spécifier où les coupures publicitaires doivent apparaître.
  5. Collez votre tag d'emplacement publicitaire, en ajoutant votre chaîne de requête des paramètres de ciblage des annonces.
  6. Cliquez sur sauvegarder.
  7. Recherchez votre nouvelle configuration d'annonce dans le tableau et copiez votre ID de configuration d'annonce. Vous aurez besoin de cette valeur pour la configuration de la publicité Beacon.

Marches balises - VOD

  1. Ouvrez l'interface utilisateur Beacon Classic et accédez au Publicité languette.
  2. Cliquez sur Ajouter une nouvelle configuration.
  3. Nommez la configuration de l'annonce.
  4. Pour le Fournisseur sélectionner Brightcove SSAI.
  5. Collez l'URL de ciblage publicitaire créée à partir de la section ci-dessus.
  6. Coller dans le ID de configuration d'annonce SSAI copié dans les étapes ci-dessus dans ID de configuration d'annonce VOD.
  7. Cliquez sur Créer une nouvelle publicité.

Étapes en direct

Étapes de Video Cloud - Live

  1. Dans Video Cloud Studio, accédez à Admin → Paramètres d'annonce côté serveur. (Pour plus de détails, consultez le Mise en œuvre de la VOD SSAI document.)
  2. Du Créer une configuration d'annonce liste déroulante, sélectionnez Habitent
  3. Ajouter un nom pour la configuration de l'annonce
  4. Sélectionner VASTE votre réponse à l'annonce
  5. Collez votre tag d'emplacement publicitaire, en ajoutant votre chaîne de requête des paramètres de ciblage des annonces.
  6. Cliquez sur sauvegarder.
  7. Recherchez votre nouvelle configuration d'annonce dans le tableau et copiez votre ID de configuration d'annonce. Vous aurez besoin de cette valeur pour la configuration de la publicité Beacon.

Étapes de balise - En direct

  1. Ouvrez l'interface utilisateur de Beacon Classic et accédez au Publicité languette.
  2. Cliquez sur Ajouter une nouvelle configuration.
  3. Nommez la configuration de l'annonce.
  4. Pour le Fournisseur sélectionner Brightcove SSAI
  5. Collez l'URL de ciblage publicitaire créée à partir de la section ci-dessus.
  6. Coller dans le ID de configuration d'annonce SSAI copié dans les étapes ci-dessus dans ID de configuration d'annonce VOD.
  7. Cliquez sur Créer une nouvelle publicité.
