    Phases d'implantation

    Dans cette rubrique, vous allez examiner les phases d'une implémentation de Brightcove Beacon.

    Introduction

    Dans ce document, un aperçu de haut niveau de l'ensemble du processus de mise en œuvre de Brightcove Beacon est fourni. Il soulignera que vous, en tant que client, avez un certain nombre de tâches à effectuer avant de rencontrer un représentant Brightcove pour créer des applications de votre appareil.

    Présentation

      Dans cette section, les phases sont introduites, brièvement définies et le résultat de la phase est indiqué. Plus loin dans le document, des listes de contrôle, s'il y a lieu, sont fournies pour vous aider à vous préparer à chaque phase.

    1. Phase 1 : Préparez-vous à la création de l'expérience Brightcove Beacon

      Plus tard dans le processus de mise en œuvre de Brightcove Beacon (Phase 4), vous construirez l'expérience Brightcove Beacon. Vous le ferez en travaillant avec un représentant de Brightcove. Pour le moment, vous aurez besoin d'avoir des artefacts de conception prêts, ainsi que des décisions prises sur les plateformes sur lesquelles vous souhaitez déployer, stratégie de monétisation, stratégie publicitaire, etc.

      RÉSULTAT FINAL : Un ensemble de ressources et de stratégies définies pour une rencontre avec un représentant de Brightcove afin de créer l'expérience Brightcove Beacon.

    2. Phase 2 : Établir des relations avec des fournisseurs tiers

      Il existe de nombreuses relations avec des fournisseurs tiers que vous devez établir avant de rencontrer un représentant de Brightcove pour créer l'expérience Brightcove Beacon. Il s'agit notamment de contacter les fournisseurs pour les plateformes sur lesquelles vous choisissez de déployer une application, un fournisseur de serveur publicitaire, Stripe pour les paiements en ligne si vous lancez des applications Web et/ou Smart TV, etc. Il est TRÈS IMPORTANT que ce processus démarre le plus tôt possible afin de ne pas causer de retards au moment de la soumission de l'application de l'appareil.

      RÉSULTAT FINAL : Comptes créés avec tous les fournisseurs tiers souhaités pour être utilisés lors d'une rencontre avec un représentant Brightcove afin de créer l'expérience Brightcove Beacon.

    3. Phase 3 : Créez votre conception d'application

      Deux parties de la conception de votre application sont faites ici. La première est de planifier quelles vidéos seront disponibles pour les clients et comment les regrouper et les commander. Essentiellement, votre architecture de l'information. La deuxième partie consiste à choisir l'apparence des écrans spécifiques de votre application, compte tenu d'un certain nombre de choix.

      RÉSULTAT FINAL : Deux feuilles de calcul, créées lors de l'utilisation des documents appropriés, sont remplies et prêtes à être soumises au personnel de Brightcove lorsque l'expérience Brightcove Beacon est prête à être créée.

    4. Phase 4 : Ingérer ou ajouter du contenu et configurer votre expérience Brightcove Beacon

      Ici, vous allez utiliser la console d'administration Brightcove Beacon pour configurer votre expérience. Ici, vous allez importer des ressources multimédia à partir de Video Cloud et ajouter d'autres contenus. La configuration de la monétisation et de la mise en page sont également effectuées. À ce stade, il n'est pas nécessaire d'ajouter tout le contenu, mais au moins un contenu représentatif pour des tests approfondis est nécessaire.

      RÉSULTAT FINAL : Le contenu est ajouté et la configuration est effectuée via Brightcove Beacon.

    5. Phase 5 : Créer des applications d'appareil à partir de votre expérience Brightcove Beacon

      Vous travaillerez avec un représentant de Brightcove pour créer l'expérience Brightcove Beacon. Le travail effectué dans les phases 1 à 4 est utilisé pour créer l'expérience. À partir de Brightcove Beacon Experience, les applications pour appareils sont conçues. À ce stade, les tests d'acceptation des utilisateurs sont également effectués pour s'assurer que les applications fonctionnent comme vous le souhaitez.

      RÉSULTAT FINAL : Les fichiers de l'application de l'appareil sont créés, testés et disponibles pour être soumis aux vendeurs/magasins souhaités.

    6. Phase 6 : Soumettre votre demande d'appareil au (s) magasin (s) correspondant (s)

      À ce stade, vous avez les applications pour vos plates-formes désirées en main, ainsi que la configuration effectuée, de sorte que vous pouvez maintenant soumettre les demandes.

      RÉSULTAT FINAL : Les applications de l'appareil sont soumises de manière appropriée et les utilisateurs commencent à utiliser les applications sous approbation du fournisseur.

    Calendrier des phases

    Notez que les phases 1 et 2 peuvent et doivent être exécutées simultanément.

    Bien sûr, une fois les applications d'appareil lancées, vous pouvez continuer à configurer et à ajouter du contenu à Brightcove Beacon sans avoir à soumettre à nouveau vos applications à leurs magasins respectifs.

    Détails - Phase 1 : Préparez-vous à la création de l'expérience Brightcove Beacon

    Voici les tâches à effectuer pour la phase 1 :

    Déterminer les plates-formes d'applications requises et les plates-formes prioritaires
    Déterminer la stratégie de monétisation
    Déterminez si des services d'authentification utilisateur personnalisés sont nécessaires
    Obtenir vos couleurs et logos de marque pour développer l'expérience et pour connaître les spécifications de l'appareil
    Assister à une formation Video Cloud - Webinaire ou formation à la demande
    Ingérer des ressources multimédia dans Video Cloud ; vous aurez besoin d'un minimum de 40 vidéos et créez 3 à 5 playlists qui utilisent ces vidéos
    Créez les champs personnalisés requis dans Video Cloud ; reportez-vous au document Champs personnalisés Video Cloud pour plus de détails
    Déterminer les exigences d'analyse
    Déterminer les géo-emplacements restreints, le cas échéant

    Détails - Phase 2 : Établir des relations avec des fournisseurs tiers

    Voici les tâches à effectuer pour la phase 2 :

    Déterminer la stratégie publicitaire et établir une relation avec le fournisseur approprié
    Établissez une relation avec Stripe pour les paiements en ligne, si vous lancez des applications Web ou Smart TV. D'autres plateformes ont un processus de paiement intégré.
    Configurer les comptes de magasin et confirmer les informations d'identification. Les fournisseurs pourraient inclure, sans s'y limiter :
    • Plateforme développeur Apple
    • Plateforme Google Developer
    • Plateforme de développeur Amazon
    • Plateforme de développement Roku
    • Autres plates-formes de développement (par exemple, Samsung, Panasonic)
    Pour plus d'informations et les URL correspondantes, reportez-vous à la section Soumettre des applications de périphérique aux magasins .

    Détails - Phase 3 : Créez votre conception d'application

    Vous devez réfléchir à la façon dont vous voulez que vos applications d'appareil apparaissent à vos clients. Dans cette phase, vous devriez travailler sur les deux documents :

    • Création de la conception de votre application - Architecture de l'information
    • Création de votre conception d'application - Options de mise en page

    Après avoir travaillé sur ces deux documents, vous aurez deux feuilles de calcul, une créée dans chaque document, que vous donnerez au personnel de Brightcove lorsqu'il sera temps de créer l'expérience Brightcove Beacon (Phase 4). Les informations contenues dans les deux feuilles de calcul détaillent les vidéos présentées aux utilisateurs, y compris la façon dont elles sont regroupées et classées. Des mises en page d'écran spécifiques sont également sélectionnées.

    Détails - Phase 4 : Ingérer ou ajouter du contenu et configurer votre expérience Brightcove Beacon

    Vous avez peut-être effectué d'autres tests et configurations, mais dans cette phase, vous voulez terminer la configuration. Vous n'avez pas besoin d'ajouter TOUS les contenus, mais à des fins de test, assurez-vous que vous disposez d'un contenu représentatif pour tous les types de médias que vous utiliserez. Vous utiliserez la console d'administration Brightcove Beacon Experience pour effectuer une configuration ultérieure.

    Détails - Phase 5 : Créer des applications d'appareil à partir de votre expérience Brightcove Beacon

    Cette phase a lieu avec l'aide du personnel de Brightcove. Les actifs, les relations avec les fournisseurs, les stratégies définies, etc. des trois premières phases sont utilisés par le représentant de Brightcove pour construire et effectuer la configuration initiale de l'expérience Brightcove Beacon. À partir de cette collaboration, les fichiers réels de l'application de l'appareil sont générés à partir de Brightcove Beacon. Aussi dans cette phase, vous pouvez effectuer des tests d'acceptation sur les applications de l'appareil.

    Détails - Phase 6 : Soumettre votre demande d'appareil au (s) magasin (s) correspondant (s)

    Pour chaque fournisseur pour lequel vous disposez d'une application d'appareil, vous devez soumettre l'application au magasin correspondant. Au cours de la phase 2, vous avez déjà établi une relation avec la plate-forme du fournisseur. Dans cette phase, vous soumettez l'application pour commencer le processus d'approbation. Dans le document Soumettre des applications de périphériques aux magasins , vous trouverez des étapes spécifiques et des liens vers la documentation fournisseur appropriée pour obtenir de l'aide dans le processus de soumission.
