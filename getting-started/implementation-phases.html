Phases de mise en œuvre

Dans cette rubrique, vous examinerez les phases d'une implémentation de Brightcove Beacon.

introduction

Dans ce document, un aperçu de haut niveau de l'ensemble du processus de mise en œuvre de Brightcove Beacon est fourni. Il soulignera qu'en tant que client, vous avez un certain nombre de tâches à effectuer avant de rencontrer un représentant Brightcove pour créer vos applications pour appareils.

Aperçu

    Dans cette section, les phases sont introduites, brièvement définies et le résultat de la phase est indiqué. Plus loin dans le document, des listes de contrôle, le cas échéant, sont fournies pour vous aider à vous préparer à chaque phase.

  1. La phase 1: Préparez-vous à la création d'une expérience Brightcove Beacon

    Plus tard dans le processus de mise en œuvre de Brightcove Beacon (Phase 4), vous créerez l'expérience Brightcove Beacon. Vous le ferez en travaillant avec un représentant Brightcove. À ce stade, vous devrez préparer certains artefacts de conception, ainsi que les décisions prises sur les plates-formes sur lesquelles vous souhaitez déployer, la stratégie de monétisation, la stratégie publicitaire, etc.

    RÉSULTAT FINAL : Une collection de ressources et de stratégies définies à utiliser lors d'une rencontre avec un représentant Brightcove pour créer l'expérience Brightcove Beacon.

  2. Phase 2: Établir des relations avec des fournisseurs tiers

    Il existe de nombreuses relations avec des fournisseurs tiers que vous devez établir avant de rencontrer un représentant Brightcove pour créer l'expérience Brightcove Beacon. Cela inclurait de contacter les fournisseurs des plates-formes sur lesquelles vous choisissez de déployer une application, un fournisseur de serveur publicitaire, Stripe pour les paiements en ligne si vous lancez des applications Web et/ou Smart TV, etc. Il est TRÈS IMPORTANT que ce processus commence le plus tôt possible afin de ne pas entraîner de retards au moment de la soumission de l'application de l'appareil.

    RÉSULTAT FINAL : Comptes créés avec tous les fournisseurs tiers souhaités à utiliser lors de la rencontre avec un représentant Brightcove pour créer l'expérience Brightcove Beacon.

  3. Étape 3 : Créez votre conception d'application

    Deux parties de la conception de votre application sont effectuées ici. La première consiste à planifier quelles vidéos seront disponibles pour les clients et comment les regrouper et les commander. Essentiellement, votre architecture de l'information. La deuxième partie consiste à choisir à quoi doivent ressembler les écrans spécifiques de votre application, en fonction d'un certain nombre de choix.

    RÉSULTAT FINAL : Deux feuilles de calcul, créées lors de l'utilisation des documents appropriés, sont remplies et prêtes à être soumises au personnel de Brightcove lorsque Brightcove Beacon Experience est prête à être créée.

  4. Étape 4 : Ingérez ou ajoutez du contenu et configurez votre expérience Brightcove Beacon

    Ici, vous utiliserez la console d'administration Brightcove Beacon pour configurer votre expérience. Ici, vous importerez des ressources multimédias à partir de Video Cloud et ajouterez d'autres contenus. La configuration de la monétisation et la mise en page sont également effectuées. À ce stade, TOUT le contenu n'a pas besoin d'être ajouté, mais au moins un contenu représentatif pour des tests approfondis est nécessaire.

    RÉSULTAT FINAL : Le contenu est ajouté et la configuration est effectuée via Brightcove Beacon.

  5. Étape 5 : Créez des applications de périphérique à partir de votre expérience Brightcove Beacon

    Vous travaillerez avec un représentant de Brightcove pour créer l'expérience Brightcove Beacon. Le travail effectué dans les phases 1 à 4 est utilisé pour créer l'expérience. À partir de Brightcove Beacon Experience, les applications de l'appareil sont créées. À ce stade, des tests d'acceptation par l'utilisateur sont également effectués pour s'assurer que les applications fonctionnent comme vous le souhaitez.

    RÉSULTAT FINAL : Les fichiers d'application de l'appareil sont créés, testés et disponibles pour être soumis aux fournisseurs/magasins souhaités.

  6. Étape 6 : Soumettez votre/vos application(s) d'appareil au(x) magasin(s) correspondant(s)

    À ce stade, vous avez les applications pour vos plates-formes souhaitées en main, ainsi que la configuration effectuée, vous pouvez donc maintenant soumettre les applications.

    RÉSULTAT FINAL : Les applications de l'appareil sont soumises de manière appropriée et les téléspectateurs commencent à utiliser les applications après approbation du fournisseur.

Chronométrage des phases

Notez que les phases 1 et 2 peuvent et doivent être exécutées simultanément.

Bien entendu, une fois les applications de l'appareil lancées, vous pouvez continuer à configurer et ajouter du contenu à Brightcove Beacon sans avoir à resoumettre les applications de votre appareil à leurs magasins respectifs.

Détails - Phase 1 : Préparez-vous à la création d'une expérience Brightcove Beacon

Voici les tâches à accomplir pour la phase 1 :

Déterminer les plates-formes d'applications d'appareil requises et les plates-formes prioritaires
Déterminer la stratégie de monétisation
Déterminer si des services d'authentification utilisateur personnalisés sont nécessaires
Obtenez les couleurs et logos de votre marque pour créer l'expérience et pour les spécifications de l'appareil
Assister à une formation Video Cloud - Webinaire ou formation à la demande
Ingérer des ressources multimédias dans Video Cloud ; vous aurez besoin d'un au moins 40 vidéos , et créer 3 à 5 listes de lecture qui utilisent ces vidéos
Créez les champs personnalisés requis dans Video Cloud ; voir le Champs personnalisés Video Cloud document pour plus de détails
Déterminer les exigences d'analyse
Déterminer les géolocalisations restreintes, le cas échéant

Détails - Phase 2 : Établir des relations avec des fournisseurs tiers

Voici les tâches à accomplir pour la phase 2 :

Déterminer la stratégie publicitaire et établir une relation avec le fournisseur approprié
Établissez une relation avec Stripe pour les paiements en ligne, si vous lancez des applications pour appareils Web ou Smart TV. D'autres plateformes ont un processus de paiement intégré.
Configurez les comptes de magasin et confirmez les informations d'identification. Les fournisseurs peuvent inclure, sans s'y limiter :
  • Plateforme de développeur Apple
  • Plateforme de développeur Google
  • Plateforme de développeur Amazon
  • Plateforme de développement Roku
  • Autres plates-formes de développement (par exemple Samsung)
Voir le Soumission d'applications d'appareils aux magasins pour plus d'informations et les URL correspondantes pour démarrer ce processus.

Détails - Phase 3 : Créez votre conception d'application

Vous devez réfléchir à la manière dont vous souhaitez que vos applications pour appareils apparaissent à vos clients. Dans cette phase, vous devez travailler à travers les deux documents :

  • Création de votre conception d'application - Architecture de l'information
  • Création de votre conception d'application - Options de mise en page

Après avoir parcouru ces deux documents, vous disposerez de deux feuilles de calcul, une créée dans chaque document, que vous remettrez au personnel de Brightcove lorsqu'il sera temps de créer Brightcove Beacon Experience (Phase 4). Les informations contenues dans les deux feuilles de calcul détaillent les vidéos présentées aux utilisateurs, y compris la manière dont elles sont regroupées et classées. Des dispositions d'écran spécifiques sont également sélectionnées.

Détails - Phase 4 : Ingérez ou ajoutez du contenu et configurez votre expérience Brightcove Beacon

Vous avez peut-être effectué d'autres tests et configurations, mais dans cette phase, vous souhaitez terminer la configuration. Vous n'êtes pas obligé d'ajouter TOUT le contenu, mais à des fins de test, assurez-vous d'avoir un contenu représentatif pour tous les types de médias que vous utiliserez. Vous utiliserez la console d'administration Brightcove Beacon Experience pour effectuer une configuration supplémentaire.

Détails - Phase 5 : Créez des applications de périphérique à partir de votre expérience Brightcove Beacon

Cette phase se déroule avec l'aide du personnel de Brightcove. Les actifs, les relations avec les fournisseurs, les stratégies définies, etc. des trois premières phases sont utilisés par le représentant de Brightcove pour créer et effectuer la configuration initiale de Brightcove Beacon Experience. À partir de cette collaboration, les fichiers d'application de périphérique réels sont générés à partir de Brightcove Beacon. Également dans cette phase, vous pouvez effectuer des tests d'acceptation sur les applications de l'appareil.

Détails - Phase 6 : Soumettez votre/vos application(s) d'appareil au(x) magasin(s) correspondant(s)

Pour chaque fournisseur pour lequel vous avez une application d'appareil, vous devez soumettre l'application au magasin correspondant. Au cours de la phase 2, vous avez précédemment établi une relation avec la plate-forme du fournisseur. Dans cette phase, vous soumettez donc l'application pour commencer le processus d'approbation. Dans le Soumission d'applications d'appareils aux magasins document, vous trouverez des étapes spécifiques et des liens vers la documentation appropriée du fournisseur pour vous aider dans le processus de soumission.
